# Gantry UI Redesign Worktrees

Baseline commit: `113754f` (`checkpoint: save current editor redesign`)

Goal: create four independent, functional redesigns of `skills/gantry/public/` and review them one at a time.

Directions:

1. `ui-redesign-a-command-center` - dense professional command center for approval-gate throughput.
2. `ui-redesign-b-editorial-studio` - spacious writing-first studio that makes pseudocode feel authored.
3. `ui-redesign-c-terminal-forge` - dark technical forge with strong keyboard/operator energy.
4. `ui-redesign-d-canvas-board` - visual workflow board focused on steps, queues, and approval progress.

Each worktree should preserve the existing server/API contract and keep edits scoped to:

- `skills/gantry/public/index.html`
- `skills/gantry/public/styles.css`
- `skills/gantry/public/app.js`
